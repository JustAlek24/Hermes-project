# Hermes-project
*Назван в честь великого рэпера из Борисоглебска*.
Проект приложения для передачи данных любого формата без использования постоянного сервера

## Идея
Сделать десктопную программу для быстрой, удобной передачи файлов другим пользователям. Этакий базовый файлообменник

## Стек технологий
Язык программирования - Python
Интерфейс - PySide6 / QML / Kivy
Пайсайд тупое старое говно, QML лучше, но не кросплатформенный, Kivy кросплатформенный, но придётся изучать с нуля.
Протокол P2P для передачи: файлов клиента + json с данными
Сценарий передачи данных: gossip

Дополнительно к репозиторию подключен бот для уведомлений https://github.com/JustAlek24/Noti-Bot

## Разбор работы (новое на 03.06.2026)

### Сценарий первого запуска

Сначала у нас есть прайм пир, вшитый в бд как первый и единственный существующий. Это наш первый экземпляр программы, от него будет всё строится. Он по сути своей какашка безрукая и безногая, т.к. ему некуда ходить и не к кому стучаться (пинговать). Он становится важен только тогда, когда появляется новый пир - второй по последовательности создания в нашем примере. Когда программа запускается вторым клиентом - она сразу начинает проверять, а прописан ли клиент как пир в базе данных. Логично, что записи об этом пире в бд не будет, но там уже будет лежать наш прайм пир. По принципу bootstrap node наш новый пир будет стучаться к прайм пиру -> получать от него весь список известных пиров -> сохранять в свою бд. Сам прайм пир, когда получит пинг от нового клиента добавит его в свою бд.
