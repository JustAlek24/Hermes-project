# Hermes-project
*Назван в честь великого рэпера из Борисоглебска*.
Проект приложения для передачи данных любого формата без использования постоянного сервера

## Идея
Сделать десктопную программу для быстрой, удобной передачи файлов другим пользователям. Этакий базовый файлообменник

## Стек технологий
Язык программирования - Python
Интерфейс - PySide6 / QML / Kivy
Пайсайд тупое старое говно, QML лучше, но не кросплатформенный, Kivy кросплатформенный, но придётся изучать с нуля.
Протокол P2P для передачи файлов клиента + json-ов с данными
Протокол передачи данных - gossip
Бд на SQLite

Дополнительно к репозиторию подключен бот для уведомлений https://github.com/JustAlek24/Noti-Bot

## Разбор работы (новое на 07.06.2026)

### Сценарий первого запуска

Сначала у нас есть прайм пир, вшитый в бд как первый и единственный существующий. Когда программа запускается вторым клиентом - она сразу начинает проверять, а прописан ли даный клиент как пир в базе данных. Логично, что записи об этом пире в бд не будет, но там уже будет лежать наш прайм пир. Наш новый пир будет стучаться к прайм пиру -> получать от него список известных пиров -> сохранять в свою бд. Сам прайм пир, когда получит пинг от нового клиента добавит его в свою бд. При следующем контакте прайм пира с кем угодно новичок разнесётся дальше.

## Minimum value product (MVP) - Минимальный рабочий продукт

Как первая рабочая версия программа должна включать:

1) БД "Peer" с полями:
-  peer_id TEXT PRIMARY KEY - уникальный айдишник пира
-  peer_name TEXT - юзернейм пользователя
-  ip TEXT - айпи адрес пира
-  port INTEGER - порт для передачи данных
-  last_seen INTEGER - дата последней активности пира timestamp
-  updated_at INTEGER - дата последнего изменения данных о пире
-  version INTEGER - версия записи в бд
